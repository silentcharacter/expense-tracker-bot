Architecture Document v4.0

Expense Tracker Telegram Bot

Голосовой учёт расходов через Gemini 3.1 Flash Lite — аудио → JSON за один вызов, мультивалютность с базой в USD, аналитика в Google Sheets

01

Обзор системы

🎯 Задача

Многопользовательский Telegram-бот для учёта расходов голосом и текстом. Gemini 3.1 Flash Lite парсит аудио в structured JSON за один вызов. Каждый пользователь получает персональный Google Spreadsheet с аналитикой. Валютно-агностичная модель: локальная валюта + USD база.

⚡ Ключевые фичи

  • Голосовой ввод на русском/английском
  • Один API-вызов: аудио → structured JSON (Gemini)
  • Multi-user: персональный Spreadsheet на юзера
  • Автоматический онбординг через /start
  • Мультивалютность — локальная валюта + USD база
  • Google Sheets как хранилище + аналитика
  • Inline-команды для статистики
  • Фото чеков с Vision-парсингом

🧰 Стек технологий

  • Python 3.12 + asyncio
  • python-telegram-bot (async)
  • Google Gemini 3.1 Flash Lite (STT + парсинг)
  • google-genai SDK
  • Google Sheets API (gspread)
  • Google Drive API (создание Spreadsheets)
  • Google Cloud Functions (хостинг)
  • ExchangeRate API (USD конвертация)

🔄 Data Flow — Голосовой ввод

1. Telegram Voice Message

Пользователь отправляет голосовое: "Обед в кафе триста пятьдесят бат"

Telegram Bot API OGG/OPUS

2. Gemini 3.1 Flash Lite — Audio → JSON

OGG файл отправляется напрямую в Gemini 3.1 Flash Lite. Один вызов: транскрипция + извлечение amount, currency, category, description. Ответ — structured JSON через response_schema. Нет промежуточного шага.

Gemini 3.1 Flash Lite ~200ms ~$0.001/запрос

3. Currency Conversion

amount_local сохраняется как есть. amount_base рассчитывается через Exchange Rate API в base_currency юзера. Курс кешируется на 24 часа.

ExchangeRate API per-user base_currency

4. Confirmation

Бот отправляет inline-кнопки: "✅ Еда: 350 THB (10.15 USD) — обед в кафе". Суммы показываются в local_currency и base_currency юзера. Кнопки: [Ок] [Изменить категорию] [Изменить сумму] [Отмена]

InlineKeyboard CallbackQuery

5. Save to Sheets

После подтверждения — строка добавляется в персональный Google Sheets юзера. Timestamp, amount_local, local_currency, amount_base, fx_rate, категория, описание, источник.

gspread Google Sheets API
03

Multi-User архитектура

👥 Принцип

Каждый пользователь получает персональный Google Spreadsheet, автоматически созданный из шаблона при первом /start. Данные полностью изолированы. Master-реестр (отдельный Spreadsheet) хранит маппинг telegram_id → spreadsheet_id.

🚀 Onboarding Flow — путь нового пользователя

1. /start

Вася нажимает /start. Бот проверяет master-реестр — telegram_id не найден, значит новый пользователь.

User Registry

2. Выбор валют

Бот спрашивает базовую валюту (для аналитики и бюджетов) и дефолтную валюту (подставляется когда валюта не указана). Inline-кнопки с популярными вариантами + ручной ввод. Валидация по ISO 4217 — бот принимает только реальные коды валют.

InlineKeyboard ISO 4217 validation

3. Создание Spreadsheet

Бот копирует template через Drive API. Копия именуется "Expenses — Вася". Ownership передаётся admin-аккаунту. Service Account остаётся editor'ом.

Drive API files().copy()

4. Регистрация в реестре

В master-реестр записывается: telegram_id, имя, spreadsheet_id, base_currency, default_currency, дата создания. Кеш обновляется.

Master Registry gspread

5. Приветствие

"Готово! Базовая валюта: USD, дефолтная: THB. Отправляй голосовые или текстом. Для доступа к таблице — /email your@gmail.com"

Telegram Bot API

6. (Опционально) /email

Вася пишет: /email vasya@gmail.com. Бот расшаривает Spreadsheet и передаёт ownership Васе. Теперь файл живёт в Васином Google Drive. Вася — полный хозяин данных.

Drive API permissions().create() transferOwnership

⚙️ User Registry Service

from googleapiclient.discovery import build

class UserRegistry:
    def __init__(self, drive, sheets, config):
        self.drive = drive
        self.sheets = sheets
        self.cache = {}  # telegram_id → spreadsheet_id

    async def get_or_create(self, telegram_id, name):
        # Проверяем кеш → реестр → создаём
        if telegram_id in self.cache:
            return self.cache[telegram_id]

        user = self.find_in_registry(telegram_id)
        if user:
            self.cache[telegram_id] = user.spreadsheet_id
            return user.spreadsheet_id

        return await self.create_user(telegram_id, name)

    async def create_user(self, telegram_id, name,
                          base_cur, default_cur):
        # 0. Валидация валют
        if not self.validate_currency(base_cur):
            raise ValueError(f"Invalid: {base_cur}")
        if not self.validate_currency(default_cur):
            raise ValueError(f"Invalid: {default_cur}")

        # 1. Копируем template
        copy = self.drive.files().copy(
            fileId=TEMPLATE_ID,
            body={"name": f"Expenses — {name}",
                  "parents": [FOLDER_ID]}
        ).execute()

        # 2. Ownership → admin
        self.drive.permissions().create(
            fileId=copy['id'],
            transferOwnership=True,
            body={"type": "user",
                  "role": "owner",
                  "emailAddress": ADMIN_EMAIL}
        ).execute()

        # 3. Записываем в реестр
        self.sheets.append_row(REGISTRY_ID, [
            telegram_id, name, copy['id'],
            "admin", "",
            base_cur, default_cur,
            datetime.now()
        ])

        self.cache[telegram_id] = copy['id']
        return copy['id']

    async def transfer_to_user(self, telegram_id, email):
        # /email command → transfer ownership
        sheet_id = self.cache[telegram_id]
        self.drive.permissions().create(
            fileId=sheet_id,
            transferOwnership=True,
            body={"type": "user",
                  "role": "owner",
                  "emailAddress": email}
        ).execute()

    # ISO 4217 валидация
    VALID_CURRENCIES = {
        "USD", "EUR", "GBP", "THB",
        "ILS", "GEL", "JPY", "CNY",
        "RUB", "TRY", "BRL", "INR",
        # ... все ISO 4217 коды
    }

    def validate_currency(self, code: str) -> bool:
        return (
            code.upper() in self.VALID_CURRENCIES
            and len(code) == 3
            and code.isalpha()
        )
📐 Масштаб: Template Spreadsheet ~50KB. При 100 пользователях — ~5MB из 15GB лимита Service Account (до transfer ownership). API quota: 300 req/min на проект — при 100 юзерах по 30 транзакций/день = ~3000 req/день, далеко от лимита. Регистрация нового юзера — 3 API-вызова (copy + permission + registry write), занимает ~2 секунды.

Gemini Prompt Design

🧠 Gemini 3.1 Flash Lite — System Instruction + Schema

# Gemini 3.1 Flash Lite: аудио → structured JSON за один вызов

import google.generativeai as genai
from pydantic import BaseModel

class Expense(BaseModel):
    amount: float        # сумма
    currency: str       # ISO 4217 (THB, USD, EUR...)
    category: str       # food, transport, housing...
    subcategory: str    # restaurant, grocery, fuel...
    description: str    # краткое описание 2-5 слов

model = genai.GenerativeModel(
    "gemini-3.1-flash-lite-preview",
    system_instruction="""
Ты — парсер расходов. Прослушай аудио и извлеки:
- amount: число
- currency: ISO 4217 ("бат/baht" → THB,
  "долларов/$" → USD, "евро/€" → EUR)
- Если валюта не указана → {user_default_currency}
- category: food|transport|housing|health|
  entertainment|shopping|education|services|
  subscriptions|travel|other
- subcategory: уточнение
- description: краткое описание на русском
""".format(user_default_currency=user.default_currency),
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=Expense
    )
)

# Один вызов — аудио на входе, типизированный JSON на выходе
audio = genai.upload_file("voice.ogg")
response = model.generate_content(audio)
expense = Expense.model_validate_json(response.text)

# expense.amount=350, expense.currency="THB", ...
💡 Почему Gemini 3.1 Flash Lite: Принимает аудио нативно + возвращает guaranteed JSON через response_schema. Один вызов вместо двух (Whisper + LLM). $0.25/M input, $1.50/M output (~$0.001/запрос). 2.5x быстрее 2.5 Flash. Free tier: 500 RPD. Улучшенное качество ASR.

🎙 Voice Handler

async def handle_voice(update, context):
    # 0. Получить настройки юзера
    user = registry.get(update.user.id)

    # 1. Скачать OGG файл
    voice = update.message.voice
    file = await voice.get_file()
    buf = await file.download_as_bytearray()

    # 2. Gemini: аудио → JSON (один вызов)
    expense = await gemini.parse_audio(
      buf, default_currency=user.default_currency
    )

    # 3. Конвертировать в base_currency
    fx = await currency.get_rate(
      expense.currency, user.base_currency
    )
    expense.amount_base = expense.amount * fx
    expense.fx_rate = fx

    # 4. Показать подтверждение
    await send_confirmation(
      update, expense, user
    )

Google Sheets — структура

📑 Листы (Sheets)

Transactions

Основной лист. Каждая строка — одна транзакция. Все колонки из модели данных выше. Сортировка по timestamp desc.

Dashboard

Сводные формулы: SUMIFS по категориям и периодам, pivot tables, графики. Обновляется автоматически при добавлении строк.

Categories

Справочник категорий и подкатегорий. Редактируется руками — бот читает при старте. Бюджеты по категориям.

Exchange Rates

Кеш курсов валют. Обновляется раз в день. GOOGLEFINANCE формулы для real-time курсов.

📈 Dashboard формулы

// Расходы за текущий месяц по категории (в base_currency)
=SUMIFS(Transactions!F:F,
  Transactions!B:B, ">="&DATE(YEAR(TODAY()),MONTH(TODAY()),1),
  Transactions!H:H, "food")

// Средний дневной расход за последние 30 дней (base_currency)
=SUMIFS(Transactions!F:F,
  Transactions!B:B, ">="&TODAY()-30) / 30

// Топ-5 категорий по amount_base
=QUERY(Transactions!A:L,
  "SELECT H, SUM(F) WHERE B >= date '"
  &TEXT(EOMONTH(TODAY(),-1)+1,"yyyy-MM-dd")
  &"' GROUP BY H ORDER BY SUM(F) DESC LIMIT 5")

// Расходы по валютам (локальные суммы)
=QUERY(Transactions!A:L,
  "SELECT E, SUM(D) WHERE B >= date '"
  &TEXT(EOMONTH(TODAY(),-1)+1,"yyyy-MM-dd")
  &"' GROUP BY E ORDER BY SUM(F) DESC")

⚡ Entry Point (main.py)

import functions_framework
from telegram import Update, Bot

@functions_framework.http
def webhook(request):
    """Telegram webhook → Cloud Function"""
    bot = Bot(token=TELEGRAM_TOKEN)
    update = Update.de_json(
        request.get_json(), bot
    )

    # Route by message type
    if update.message.voice:
        return handle_voice(update)
    elif update.message.text:
        return handle_text(update)
    elif update.message.photo:
        return handle_photo(update)

    return "ok", 200

⏱ Cold Start и Latency

Cloud Functions 2nd gen имеет холодный старт ~1-2 сек после простоя. Для бота это некритично — пользователь и так ожидает ответ. Типичный flow:

ЭтапВремяПримечание
Cold start (если был простой) ~1-2 сек Только первый запрос после ~15 мин простоя
Telegram → Cloud Function ~50ms Webhook, Сингапур → Сингапур
Скачать аудио из Telegram ~100ms OGG файл, обычно 10-50KB
Gemini 3.1 Flash Lite (audio → JSON) ~150-300ms Один вызов, structured output
Exchange Rate (из кеша) ~5ms In-memory кеш, обновление раз в 24ч

Ответ в Telegram ~50ms Inline keyboard с подтверждением

Безопасность

🔒 Меры защиты

  • Данные изолированы: каждый юзер — отдельный Spreadsheet
  • Открытая регистрация через /start (или whitelist через ALLOWED_USER_IDS)
  • GCP Default Service Account — editor-доступ к Spreadsheets
  • Env vars в .env.yaml + Secret Manager (для production)
  • Webhook с secret token для верификации
  • Cloud Functions IAM — контроль кто может вызвать функцию

🛡 Privacy

  • Голосовые файлы удаляются сразу после обработки Gemini 3.1 FL
  • Gemini 3.1 FL получает только аудио, без метаданных пользователя
  • Ownership Spreadsheet передаётся юзеру (при наличии email)
  • Admin не видит данные юзеров без явного доступа
  • Нет хранения на сервере бота (stateless)
