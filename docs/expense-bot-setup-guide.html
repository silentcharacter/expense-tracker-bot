Setup Guide

Expense Tracker Bot — Setup

Пошаговая инструкция: от нуля до работающего бота. Все аккаунты, ключи и конфигурация за ~20 минут.

00

Что понадобится

  • gcloud CLI (cloud.google.com/sdk/docs/install)
  • ~20 минут времени

01

Google Cloud Project

~5 мин

☁️ Создание проекта

1

Открой Google Cloud Console

Перейди на console.cloud.google.com и залогинься своим Gmail.

2

Создай новый проект

Кликни на выпадающий список проектов сверху → "New Project". Имя: expense-bot. Organization оставь пустым если нет Workspace.

3

Или через gcloud CLI

# Авторизация
gcloud auth login

# Создание проекта
gcloud projects create expense-bot-12345 \
  --name="Expense Bot"

# Установить как текущий
gcloud config set project expense-bot-12345

ID проекта должен быть уникальным глобально — добавь цифры к имени.

💰 Billing: Google Cloud требует привязку карты, но для нашего проекта всё попадает в бесплатный tier. Новые аккаунты получают $300 кредита на 90 дней. Cloud Functions и Firestore бесплатны в наших объёмах.

🔌 Включение API

Включи четыре API в проекте:

# Все API одной командой
gcloud services enable \
  firestore.googleapis.com \
  cloudfunctions.googleapis.com \
  generativelanguage.googleapis.com

# Проверить что включены
gcloud services list --enabled

Или через UI: APIs & Services → Library → ищи каждый по имени → Enable.

Gemini API Key

~2 мин

🧠 Получение ключа через AI Studio

1

Открой AI Studio

Перейди на aistudio.google.com (тот же Gmail-аккаунт).

2

Создай API Key

Кликни "Get API Key" → "Create API Key in existing project" → выбери expense-bot.

3

Скопируй ключ

Ключ выглядит как AIzaSy.... Сохрани его — это GOOGLE_API_KEY в .env.yaml.

🎁 Free tier Gemini 3.1 Flash Lite: 15 RPM, 250K TPM, 500 RPD. Для персонального бота (30 транзакций/день) — бесплатно. Хватит на ~16 активных пользователей. При большей нагрузке — включи billing (Tier 1), стоимость: $0.25/M input, $1.50/M output (~$0.90/мес при 30 запросах/день).
✓

Проверка

curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-lite-preview:generateContent?key=YOUR_API_KEY" \
  -H 'Content-Type: application/json' \

03

Firestore + Application Default Credentials

~5 мин

🔥 Настройка Firestore

1

Включи Firestore API

gcloud services enable firestore.googleapis.com \
  --project=expense-bot-12345
2

Создай базу данных Firestore

gcloud firestore databases create \
  --project=expense-bot-12345 \
  --location=asia-southeast1 \
  --type=firestore-native
3

Создай сервис-аккаунт и выдай доступ к Firestore

# Создать сервис-аккаунт для Cloud Function
gcloud iam service-accounts create expense-bot-sa \
  --display-name="Expense Bot Service Account"

# Выдать доступ к Firestore
gcloud projects add-iam-policy-binding expense-bot-12345 \
  --member="serviceAccount:expense-bot-sa@expense-bot-12345.iam.gserviceaccount.com" \
  --role="roles/datastore.user"

Cloud Function будет запускаться от этого сервис-аккаунта — credentials.json не нужен.

4

Настрой Application Default Credentials для локальной разработки

# Один раз — авторизует gcloud для локальных вызовов GCP API
gcloud auth application-default login

Credentials сохранятся в ~/.config/gcloud/ и подхватятся автоматически. Файл credentials.json не нужен.

✅ Никаких JSON-ключей! Cloud Functions использует сервис-аккаунт автоматически. Локальная разработка — через ADC.

06

Сборка .env.yaml

~2 мин

⚙️ Финальная конфигурация

Собери все ключи в один файл:

# .env.yaml — конфигурация Cloud Function

# Telegram
TELEGRAM_BOT_TOKEN: "7123456789:AAH..."

# Gemini
GOOGLE_API_KEY: "AIzaSy..."

# Exchange Rate
EXCHANGE_RATE_API_KEY: "abc123..."

# Storage
STORAGE_BACKEND: "firestore"

# App Config
TIMEZONE: "Asia/Bangkok"
🔒 Не коммить .env.yaml! Добавь в .gitignore. Для локальной разработки используй .env.yaml. Для продакшена — Cloud Functions environment variables или Secret Manager. 
# .gitignore
.env.yaml
__pycache__/
*.pyc

08

Деплой на Cloud Functions

~5 мин

🚀 Деплой функции

1

Включи дополнительные API

Cloud Functions Gen2 требует несколько API помимо тех, что мы включили ранее:

gcloud services enable \
  cloudbuild.googleapis.com \
  artifactregistry.googleapis.com \
  run.googleapis.com \
  storage.googleapis.com
2

Деплой

Из корня проекта запусти:

gcloud functions deploy expense-bot \
  --gen2 \
  --runtime=python312 \
  --region=asia-southeast1 \
  --source=. \
  --entry-point=webhook \
  --trigger-http \
  --allow-unauthenticated \
  --env-vars-file=.env.yaml \
  --memory=256MB \
  --timeout=60s \
  --service-account=expense-bot-sa@PROJECT_ID.iam.gserviceaccount.com
⚠️ --service-account: Обязательно укажи Service Account, созданный в шаге 03. Без этого флага Cloud Function будет работать от дефолтного Compute SA, у которого нет доступа к расшаренным таблицам. Замени PROJECT_ID на ID своего проекта.
3

Получи URL функции

После успешного деплоя в выводе будет URL вида:

https://asia-southeast1-PROJECT_ID.cloudfunctions.net/expense-bot

Это URL для Telegram webhook.

4

Установи Telegram webhook

curl "https://api.telegram.org/botTELEGRAM_BOT_TOKEN/setWebhook?url=FUNCTION_URL"

Замени TELEGRAM_BOT_TOKEN и FUNCTION_URL на свои значения. Ответ должен содержать "ok": true.

✓

Проверка

Открой бота в Telegram и отправь /start. Бот должен ответить приветствием и зарегистрировать тебя.

🔄 Обновление (редеплой)

При изменении кода просто запусти ту же команду деплоя повторно. При изменении только переменных окружения — достаточно обновить .env.yaml и задеплоить заново.

# Проверить логи после деплоя
gcloud functions logs read expense-bot \
  --region=asia-southeast1 \
  --gen2 \
  --limit=50
🎉 Бот запущен! Отправь голосовое или текстовое сообщение с расходом — бот распарсит его через Gemini и запишет в Google Sheets.
09

Cloud Scheduler (Recurring Expenses)

~5 мин

🔁 Как работает

Cloud Scheduler ежедневно вызывает эндпоинт /cron/recurring на Cloud Function. Функция проходит по всем активным пользователям, находит шаблоны с совпадающим day_of_month и материализует их в лист Transactions. Операция идемпотентна — повторный запуск в тот же день пропустит уже существующие записи.

⚙️ Настройка

1

Добавь CRON_SECRET в .env.yaml

Придумай любой секрет — он защищает эндпоинт от несанкционированного вызова:

CRON_SECRET: "your-strong-secret-here"

Передеплой функцию после изменения:

./deploy.sh
2

Включи Cloud Scheduler API

gcloud services enable cloudscheduler.googleapis.com \
  --project=expense-bot-489609
3

Создай задание

gcloud scheduler jobs create http expense-bot-recurring \
  --project=expense-bot-489609 \
  --location=asia-southeast1 \
  --schedule="0 9 * * *" \
  --uri="https://asia-southeast1-expense-bot-489609.cloudfunctions.net/expense-bot/cron/recurring" \
  --http-method=POST \
  --headers="X-Cron-Secret=your-strong-secret-here" \
  --time-zone="Asia/Singapore" \
  --attempt-deadline=5m

0 9 * * * — каждый день в 09:00. Замени часовой пояс под себя.

✓

Проверь — запусти вручную

# Принудительный запуск
gcloud scheduler jobs run expense-bot-recurring \
  --project=expense-bot-489609 \
  --location=asia-southeast1

# Посмотреть логи
gcloud functions logs read expense-bot \
  --project=expense-bot-489609 \
  --region=asia-southeast1 \
  --limit=50

В логах появится строка вида: Recurring cron complete: {'users': 2, 'inserted': 1, 'skipped': 0, 'errors': 0}

🛠 Управление заданием

# Статус и история последних запусков
gcloud scheduler jobs describe expense-bot-recurring \
  --project=expense-bot-489609 \
  --location=asia-southeast1

# Приостановить
gcloud scheduler jobs pause expense-bot-recurring \
  --project=expense-bot-489609 \
  --location=asia-southeast1

# Возобновить
gcloud scheduler jobs resume expense-bot-recurring \
  --project=expense-bot-489609 \
  --location=asia-southeast1

# Удалить
gcloud scheduler jobs delete expense-bot-recurring \
  --project=expense-bot-489609 \
  --location=asia-southeast1

🧪 Локальное тестирование

Для отладки без деплоя — запусти локальный сервер и вызови эндпоинт напрямую:

# 1. Запусти сервер
python run_local.py

# 2. В другом терминале — вызови крон
curl -X POST http://localhost:8080/cron/recurring \
  -H "X-Cron-Secret: your-strong-secret-here"

Для теста с конкретной датой — вызови напрямую из Python:

import asyncio
from datetime import date
from jobs.recurring_cron import run_recurring_cron

asyncio.run(run_recurring_cron(today=date(2026, 4, 21)))
💰 Стоимость: Cloud Scheduler бесплатен до 3 заданий в месяц. Одно задание для крона — в бесплатном tier.
