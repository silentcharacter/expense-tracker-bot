Setup Guide

Expense Tracker Bot — Setup

Пошаговая инструкция: от нуля до работающего бота. Все аккаунты, ключи и конфигурация за ~20 минут.

00

01

Google Cloud Project

~5 мин

☁️ Создание проекта

1

Открой Google Cloud Console

Перейди на console.cloud.google.com и залогинься своим Gmail.

2

Создай новый проект

Кликни на выпадающий список проектов сверху → "New Project". Имя: expense-bot. Organization оставь пустым если нет Workspace.

3

Или через gcloud CLI

# Авторизация
gcloud auth login

# Создание проекта
gcloud projects create expense-bot-12345 \
  --name="Expense Bot"

# Установить как текущий
gcloud config set project expense-bot-12345

ID проекта должен быть уникальным глобально — добавь цифры к имени.

💰 Billing: Google Cloud требует привязку карты, но для нашего проекта всё попадает в бесплатный tier. Новые аккаунты получают $300 кредита на 90 дней. Cloud Functions, Sheets API и Drive API бесплатны в наших объёмах.

🔌 Включение API

Включи четыре API в проекте:

# Все четыре API одной командой
gcloud services enable \
  sheets.googleapis.com \
  drive.googleapis.com \
  cloudfunctions.googleapis.com \
  generativelanguage.googleapis.com

# Проверить что включены
gcloud services list --enabled

Или через UI: APIs & Services → Library → ищи каждый по имени → Enable.

02

Gemini API Key

~2 мин

🧠 Получение ключа через AI Studio

1

Открой AI Studio

Перейди на aistudio.google.com (тот же Gmail-аккаунт).

2

Создай API Key

Кликни "Get API Key" → "Create API Key in existing project" → выбери expense-bot.

3

Скопируй ключ

Ключ выглядит как AIzaSy.... Сохрани его — это GOOGLE_API_KEY в .env.yaml.

🎁 Free tier Gemini 3.1 Flash Lite: 15 RPM, 250K TPM, 500 RPD. Для персонального бота (30 транзакций/день) — бесплатно. Хватит на ~16 активных пользователей. При большей нагрузке — включи billing (Tier 1), стоимость: $0.25/M input, $1.50/M output (~$0.90/мес при 30 запросах/день).
✓

Проверка

# Быстрый тест — должен вернуть JSON-ответ
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-lite-preview:generateContent?key=YOUR_API_KEY" \
  -H 'Content-Type: application/json' \
  -d '{"contents":[{"parts":[{"text":"Say hello"}]}]}'
03

Service Account (Sheets + Drive)

~5 мин

🤖 Создание Service Account

# Создать Service Account
gcloud iam service-accounts create expense-bot-sa \
  --display-name="Expense Bot Service Account"

# Скачать JSON-ключ
gcloud iam service-accounts keys create credentials.json \
  --iam-account=expense-bot-sa@expense-bot-12345.iam.gserviceaccount.com

Файл credentials.json появится в текущей директории. Это credentials для программного доступа к Sheets и Drive.

🔒 Безопасность: Никогда не коммить credentials.json в git! Добавь в .gitignore немедленно. Для продакшена используй Google Secret Manager. 
# Добавить в .gitignore
echo "credentials.json" >> .gitignore

📊 Настройка Google Sheets

1

Создай Template Spreadsheet

Открой sheets.google.com → создай новый Spreadsheet → назови "Expense Bot — Template". Создай листы: Transactions, Dashboard, Categories.

2

Создай Master Registry Spreadsheet

Ещё один Spreadsheet → назови "Expense Bot — Registry". Один лист Users с колонками: telegram_id, username, display_name, email, spreadsheet_id, owner, base_currency, default_currency, created_at, status.

3

Расшарь оба Service Account'у

Открой каждый Spreadsheet → Share → вставь email Service Account'а (он выглядит как [email protected]) → дай роль Editor.

4

Создай папку для пользовательских Spreadsheets

В Google Drive создай папку "ExpenseBot Users". Расшарь её Service Account'у (Editor). Скопируй ID папки из URL: drive.google.com/drive/folders/FOLDER_ID.

5

Запиши ID

Из URL каждого Spreadsheet скопируй ID: docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit. Тебе нужны три ID: TEMPLATE_SHEET_ID, REGISTRY_SHEET_ID, USERS_FOLDER_ID.

04

Telegram Bot

~2 мин

🤖 Создание бота через BotFather

1

Открой @BotFather в Telegram

Найди @BotFather в Telegram и начни чат.

2

Создай бота

# Отправь BotFather:
/newbot

# BotFather спросит имя — введи:
Expense Tracker

# BotFather спросит username — введи (должен заканчиваться на "bot"):
my_expense_tracker_bot
3

Сохрани токен

BotFather вернёт токен вида 7123456789:AAH.... Это TELEGRAM_BOT_TOKEN.

4

(Опционально) Настрой команды

# Отправь BotFather:
/setcommands

# Выбери своего бота, затем отправь список:
start - Start and register
email - Link your Google account
settings - Change currencies
today - Today's expenses
week - Weekly summary
month - Monthly report
last - Last 10 expenses
undo - Delete last expense
budget - Budget status
export - Export CSV
05

Exchange Rate API

~2 мин

💱 Регистрация

1

Зарегистрируйся

Перейди на exchangerate-api.com → Sign Up (бесплатно). Подтверди email.

2

Скопируй API Key

После логина — ключ на главной странице дашборда. Это EXCHANGE_RATE_API_KEY.

✓

Проверка

# Должен вернуть JSON с курсами
curl "https://v6.exchangerate-api.com/v6/YOUR_KEY/latest/USD"
🎁 Free tier: 1500 запросов/мес. Нам нужен ~1 запрос в день (курсы кешируются на 24 часа) = ~30 запросов/мес. Запас в 50 раз.
06

Сборка .env.yaml

~2 мин

⚙️ Финальная конфигурация

Собери все ключи в один файл:

# .env.yaml — конфигурация Cloud Function

# Telegram
TELEGRAM_BOT_TOKEN: "7123456789:AAH..."

# Gemini
GOOGLE_API_KEY: "AIzaSy..."

# Google Sheets / Drive
ADMIN_EMAIL: "[email protected]"
TEMPLATE_SHEET_ID: "1aBcDe..."
REGISTRY_SHEET_ID: "1fGhIj..."
USERS_FOLDER_ID: "1kLmNo..."

# Exchange Rate
EXCHANGE_RATE_API_KEY: "abc123..."

# App Config
TIMEZONE: "Asia/Bangkok"
🔒 Не коммить .env.yaml! Добавь в .gitignore. Для локальной разработки используй .env.yaml. Для продакшена — Cloud Functions environment variables или Secret Manager. 
# .gitignore
credentials.json
.env.yaml
__pycache__/
*.pyc
